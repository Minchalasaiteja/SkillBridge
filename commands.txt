  1. Initial Setup & Environment

  # Navigate to project directory
cd c:\Users\prajw\Downloads\skillbridge

# Create a Python virtual environment (optional, but recommended)
python -m venv venv
.\venv\Scripts\Activate.ps1

# Or use Conda (if you prefer)
conda create -n skillbridge python=3.12
conda activate skillbridge

2. Install Dependencies

# Install all required packages
pip install -r requirements.txt

# Or with specific versions if needed
pip install flask flask-cors python-dotenv google-generativeai langchain pymongo mongoengine requests pandas numpy python-json-logger opentelemetry-api opentelemetry-sdk opentelemetry-exporter-jaeger opentelemetry-instrumentation-flask opentelemetry-instrumentation-requests opentelemetry-exporter-prometheus prometheus-client flask-jwt-extended passlib bcrypt deprecated flask-session pydantic-settings


3. Environment Configuration
# Create .env file in the project root with these variables:
# Example .env content:
# FLASK_HOST=0.0.0.0
# FLASK_PORT=5000
# FLASK_DEBUG=True
# MONGODB_URI=your_mongodb_connection_string
# GEMINI_API_KEY=your_gemini_api_key
# JWT_SECRET_KEY=your_secret_key
# JAEGER_HOST=localhost
# JAEGER_PORT=6831



4. Database Seeding

# Dry-run (preview resources without saving to DB)
python .\seeder.py

# Seed 500 resources into MongoDB
python .\seeder.py --count 500 --commit

# Custom count
python .\seeder.py --count 1000 --commit


5. Run the Application

# Start the Flask app (development mode with auto-reload)
python .\app.py

# Or with Flask CLI
flask run

# Run on specific host:port
python .\app.py --host 0.0.0.0 --port 5000


6. Test API Endpoints

# Run endpoint verification tests
python .\verify_endpoints.py

# Or run comprehensive API tests
python .\test_api.py

# Test individual endpoints with curl
curl http://localhost:5000/
curl http://localhost:5000/api/health
curl http://localhost:5000/api/status


7. Docker Deployment 

# Build and run with Docker Compose
docker compose up --build

# Run in detached mode (background)
docker compose up -d

# Stop containers
docker compose down

# View logs
docker compose logs -f app
docker compose logs -f jaeger

8. Access Points

Frontend:        http://localhost:5000/
API Health:      http://localhost:5000/api/health
API Status:      http://localhost:5000/api/status
Register User:   POST http://localhost:5000/api/auth/register
Login User:      POST http://localhost:5000/api/auth/login
Jaeger UI:       http://localhost:16686 (if using Docker)
Prometheus:      http://localhost:9090 (if using Docker)

10. Troubleshooting Commands

# Check if Flask is running on port 5000
Get-NetTcpConnection -LocalPort 5000

# Kill process using port 5000 (if needed)
Stop-Process -Id (Get-NetTcpConnection -LocalPort 5000).OwningProcess -Force

# Check Python version
python --version

# List installed packages
pip list

# Verify .env is loaded
python -c "from config import settings; print(settings.flask_host, settings.flask_port)"
